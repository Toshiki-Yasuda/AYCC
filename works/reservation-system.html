音楽教室管理システム

Piano School Management System

「教えることに集中したい」

── それは、すべてのピアノの先生の願いです。

ピアノ教室の運営には、レッスン以外にも多くの業務があります。
私たちは実際の先生方の声を調査し、日々の業務で特に負担が大きい課題を整理しました。

本システムでは、教室のホームページ制作から予約・生徒・月謝・レッスン記録の管理まで、
教室運営に必要な機能をワンストップで提供します。

振替レッスンの調整に、毎回30分以上かかる

「振替がなければどれだけ楽か」

「空いている日を聞かれて提案しても、『その日は無理です』の繰り返し」

振替の調整は「先生の空き枠の確認 → 保護者への提案 → 返答待ち → 再提案」の繰り返し。LINEで1件ずつやりとりするため、1件に30分から数時間かかることも。年間レッスンの3割が振替になるケースもあります。

→ システムでできること

保護者がアプリ上で空き枠を直接確認し、希望の日時を選択。先生はワンタップで承認するだけ。やりとりの往復がなくなり、振替調整が数分で完了します。予約・変更・キャンセルがアプリ内で完結するため、LINEでの事務連絡も大幅に減らせます。

月謝の管理が、実は一番のストレス

「催促が本当につらい」

「半年滞納に気づかなかった」

個人教室では現金手渡しが主流です。受け取りの記録が曖昧になりやすく、確認や催促は先生自身がやるしかありません。調査では53%の先生が「お金の管理が一番の悩み」と回答しています。

→ システムでできること

月謝の受領をアプリ上で記録。「誰が今月未払いか」が一覧で分かり、記憶やメモに頼る必要がなくなります。記録が残ることで、トラブルの予防にもなります。

20人分の進度を、頭の中だけで管理している

「前回何を教えたか思い出すのに時間がかかる」

「紙のレッスンノートだと振り返りが大変」

レッスンの記録は紙のノートが主流で、デジタル化されていません。生徒が増えるほど、一人ひとりの進度を把握するのが難しくなります。

→ システムでできること

レッスン後に「今日の内容」「宿題」「コメント」を記録。生徒ごとの履歴が自動で蓄積されるので、次のレッスンの準備がスムーズに。保護者も自宅からレッスン内容を確認できます。

「明日レッスンでしたっけ？」── 忘れられる予約

「連絡なしの欠席が月に1〜2件ある」

「空いた時間がもったいない」

保護者の日常は忙しく、毎週のレッスン日をうっかり忘れてしまうことがあります。特にフレックス制の場合、日時が毎回異なるため忘れやすくなります。

→ システムでできること

レッスン前日に自動でリマインダーメールを送信。「明日○時からレッスンです」という通知で、うっかり忘れを防ぎます。教室の規約もアプリ上に掲載し、入会時に保護者が確認・同意するフローで「聞いていなかった」というトラブルも未然に防ぎます。

これらの課題は、先生の努力不足ではなく、仕組みがないことが原因です。
適切なシステムがあれば、先生は本来の仕事──レッスン──に集中できるようになります。

AYCC代表はマリンバ・ピアノの演奏家であり、音楽教室の運営経験者。先生の日常を知っているからこそ、現場の業務フローに寄り添った設計ができます。教室のホームページ制作から予約・生徒・月謝管理まで、汎用の予約システムでは実現できない「ピアノ教室のための」機能をワンストップで提供します。

主な機能

管理者向け機能（7項目）
  • ● 教室ホームページ ── 教室紹介・講師紹介・レッスン料金・アクセス・ブログを備えた専用サイト
  • ● ダッシュボード ── 統計指標のリアルタイム表示、月間カレンダー、今日・明日のスケジュール
  • ● 生徒管理・招待制 ── トークンベースの招待リンク自動発行、メール1通でマイページ共有
  • ● 定期レッスン管理 ── 曜日×時間帯の固定スケジュール、欠席・振替の例外管理
  • ● 月謝管理 ── 月別×生徒別の支払い追跡、滞納アラートとリマインドメール
  • ● レッスンメモ ── 内容・宿題・次回目標を記録し生徒と共有
  • ● 設定 ── 教室情報・利用規約管理、操作ログ、パスワード変更
生徒向け機能（4項目）
  • ● レッスン予約 ── 招待リンクからログイン不要でアクセス、カレンダーから空き枠を選んで予約
  • ● マイ予約管理 ── 予約一覧の確認とキャンセル
  • ● レッスン記録閲覧 ── 先生が書いたメモ・宿題・目標をマイページから確認
  • ● 教室情報・利用規約 ── 連絡先表示と規約の確認・同意
自動化・運用（4項目）
  • ● メールリマインダー ── レッスン前日に自動配信、予約忘れを防止
  • ● 操作ログ ── すべての操作を記録、管理者・生徒でフィルタリング可能
  • ● セキュリティ ── rate limit、middleware認証、JWT、bcryptハッシュ化
  • ● レスポンシブ対応 ── PC・タブレット・スマートフォンに完全対応

使用技術

Next.js 16 React 19 TypeScript PostgreSQL Prisma ORM NextAuth.js v5 JWT Tailwind CSS shadcn/ui Nodemailer Playwright Vitest

Next.jsのフルスタック構成で、サーバーサイドレンダリングとAPI統合を実現。PostgreSQLにデータを安全に保存し、Prisma ORMによる型安全なDB操作を実装。VPS上にデプロイし、低コストかつ高い自由度で運用できます。

システム構成

フロントエンド

Next.js App Router + React Server Componentsによるモダンな構成。Tailwind CSS + shadcn/uiで洗練されたUIを実現。

バックエンド

Next.js Server Actions + API Routesでフロントとバックを統合。NextAuth.js v5によるJWT認証でセキュアなアクセス制御。

データストア

PostgreSQL + Prisma ORMによる型安全なデータ管理。13テーブル構成で教室運営に必要なデータを網羅。

自動化

Cronジョブによるリマインダー自動配信。44ユニットテスト + 43 E2Eテストによる品質保証。
