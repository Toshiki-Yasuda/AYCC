tmux チートシート

最低限の操作リファレンス

セッション管理

操作 コマンド
新規セッション作成 tmux new -s 作業名
セッション一覧 tmux ls
セッションに再接続 tmux a -t 作業名
セッションを抜ける（維持） Ctrl+b → d

ウィンドウ（タブのようなもの）

操作 キー
新規ウィンドウ Ctrl+b → c
次のウィンドウ Ctrl+b → n
前のウィンドウ Ctrl+b → p
番号で移動 Ctrl+b → 0〜9
ウィンドウ名変更 Ctrl+b → ,

ペイン（画面分割）

操作 キー
左右に分割 Ctrl+b → %
上下に分割 Ctrl+b → "
ペイン間移動 Ctrl+b → 矢印キー
ペインを閉じる exit または Ctrl+d

よく使う流れ 

# VPSにMoshで接続後、セッションに復帰
tmux a -t dev

# セッションがなければ新規作成
tmux new -s dev

tmux a でデタッチしたセッションに戻れるのが、Moshとの組み合わせで「切れない環境」を実現しているポイントです。端末を閉じても tmux a で全部元通りになります。

1日の利用フロー

自宅 → 外出 → 自宅の流れで、tmuxセッションがデバイスをまたいで維持される様子。

朝 自宅 — Windows デスクトップ 
# Mosh で VPS に接続
mosh user@vps-server

# tmux セッションを作成して作業開始
tmux new -s dev

# ウィンドウを分けて作業
#   0: コーディング（Claude Code）
#   1: テスト実行
#   2: git 操作

外出前にデタッチ

Ctrl+b → d
日中 外出先 — MacBook / iPad mini / Android 
# どのデバイスからでも同じ操作
mosh user@vps-server

# 朝のセッションにそのまま復帰
tmux a -t dev

# → ウィンドウ 0, 1, 2 がすべてそのまま残っている
# → カーソル位置も、実行中の処理もそのまま

iPad mini / Android の場合は Termux + Mosh で同様に接続。病院の待合室や移動中でも tmux a で即復帰。

帰宅前にデタッチ

Ctrl+b → d
夜 自宅 — Windows デスクトップ 
# 再び同じセッションに復帰
mosh user@vps-server
tmux a -t dev

# → 1日を通してセッションが途切れていない
# → 朝の続きからそのまま再開

ポイント: tmux セッションは VPS 上で動き続けているため、手元のデバイスが変わっても tmux a だけで全ての作業状態に復帰できます。Mosh の自動再接続と組み合わせることで、通信の切断すら意識せずに作業を継続できます。
